# 🧠 Prompt 迭代与工程化日志

本项目最核心的技术挑战在于：**如何消除大模型的幻觉，确保其输出的布局方案在前端可视化时坐标 100% 准确，且符合移动端交互规范。**

## 1. 核心破局点：从“文字描述”到“绝对坐标渲染”
*   **初期方案的痛点**：早期要求 AI 用文字描述位置（如“将微信放在左侧”），导致 UI 渲染逻辑极其脆弱，文字与实际手机模拟图经常对不上。
*   **工程化解决方案 (v2 迭代)**：废除文字布局描述，强制大模型以 JSON 格式输出包含绝对网格坐标（`row`, `col`, `span`）的数据结构。由前端 Vue 组件负责解析并计算 CSS Grid 占位，彻底解决了空间错位问题。

## 2. Prompt 演进记录
为了解决大模型“偷懒”和“常识性幻觉”问题，我对 System Prompt 进行了 4 个版本的迭代：

| 版本   | 核心改动与约束                                               | 遇到的问题                                                   | 验收结果                  |
| ------ | ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------- |
| **v1** | 初始版角色设定                                               | Dock 栏被 AI 默认填入系统 App（电话、短信、浏览器），无视用户真实习惯 | ❌ 0/4 准确率              |
| **v2** | 明确约束：“Dock 必须来自用户描述”                            | 依然存在模型依靠惯性输出默认 App 的情况                      | ❌ 3/4 准确率              |
| **v3** | 加入极端强约束：“绝对不允许出现系统默认 App”                 | Dock 栏完全正确，但 App 在网格区 (grid) 和 Dock 栏重复出现   | ✅ Dock 区通过，但逻辑异常 |
| **v4** | 新增互斥约束：“同一个 App 只能出现一次，不能同时在 grid 和 dock 中出现” + 前端增加兜底去重函数 | 所有测试用例完美闭环                                         | ✅ 4/4 完美通过            |

## 3. System Prompt 核心节选 (v4 稳定版)

你是一个手机桌面布局设计专家。你精通 iOS 和 Android 的桌面交互设计，理解用户的使用习惯和效率需求。你的任务是根据用户描述的手机使用习惯，为其设计一套最优的手机桌面布局方案。

设计原则：
1. 高频 App 放在拇指热区（屏幕下半部分）
2. 同场景 App 放在同一文件夹或相邻区域
3. 小组件应根据用户场景推荐（如通勤用户推荐地图+音乐组件）
4. 首屏不超过 24 个 App 图标
5. **Dock 栏（底部固定栏）4 个 App 必须全部来自用户描述中明确提到的 App，不允许出现任何系统默认 App（如电话、短信、浏览器、相机等）。如果用户提到的 App 不足 4 个，再从高频使用场景中补充，但绝对不能出现系统默认 App**
6. **同一个 App 只能在 grid 或 dock 中出现一次，不能同时在 grid 和 dock 中出现，也不能在 grid 中出现两次。每个 App 只出现一次。**
